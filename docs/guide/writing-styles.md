# 글쓰기 스타일 가이드

이 문서는 `codebase-skill`에 포함된 글쓰기 스타일이 각각 무엇을 위한 것인지, 어떤 글에 쓰는 것이 좋은지 빠르게 이해할 수 있도록 정리한 문서입니다.

실제 스타일 원문 파일은 여기 있습니다.

- [skills/codebase-skill/writing-styles](../../skills/codebase-skill/writing-styles)

## 가장 먼저 추천하는 선택

스타일을 잘 모르겠다면 `default`부터 시작하세요.

`default`는:

- 가장 무난한 기술 블로그 형식이고
- 특정 직군에 너무 치우치지 않으며
- 대부분의 제품/기술 글에 잘 맞습니다

## 스타일별 설명

### `default`

가장 기본이 되는 기술 블로그 스타일입니다.

이럴 때 적합합니다.

- 구현 설명
- 아키텍처 소개
- 기능 정리
- 기술 비교

### `pm`

제품 매니저 또는 창업자 관점의 글에 맞는 스타일입니다.

이럴 때 적합합니다.

- 왜 이 기능을 만들었는지 설명할 때
- 어떤 옵션 중 무엇을 선택했는지 정리할 때
- 사용자 문제, 우선순위, 트레이드오프를 설명할 때

### `research`

근거와 분석이 중요한 글에 적합합니다.

이럴 때 적합합니다.

- 논문 리뷰
- 실험 결과 정리
- 벤치마크 해석
- 한계와 리스크 분석

### `marketer`

성장, 전환, 메시지 테스트 같은 마케팅 글에 적합합니다.

이럴 때 적합합니다.

- 캠페인 회고
- 전환율 변화 설명
- 카피 테스트 정리
- 채널별 실험 결과 정리

### `designer`

디자인 케이스 스터디나 UX 판단을 설명하는 글에 적합합니다.

이럴 때 적합합니다.

- UX 개선 사례
- 화면 구조 개편 이유
- 디자인 의사결정 배경
- before / after 설명

### `novel`

서사형, 경험형 글에 적합합니다.

이럴 때 적합합니다.

- 장애 대응기
- 디버깅 여정
- 감정선이 중요한 경험담

### `podcast`

대화하듯 쉽게 풀어쓰는 글에 적합합니다.

이럴 때 적합합니다.

- 질문과 답변 형식
- 개념을 편하게 설명하고 싶을 때
- 음성처럼 읽히는 흐름을 원할 때

### `vibe`

성장, 학습, 조언 중심 글에 적합합니다.

이럴 때 적합합니다.

- 개발 공부법
- 커리어 조언
- AI 시대 학습 방식
- 주니어 대상 멘토링 글

## 커스텀 스타일을 써야 하는 경우

아래에 해당하면 기본 스타일보다 커스텀 스타일이 더 좋습니다.

- 브랜드 문체가 이미 정해져 있음
- 금지 표현이 명확함
- 제목 규칙이 따로 있음
- 문단 구조가 정형화돼 있음
- 예시 문장까지 강하게 통제하고 싶음

시작 파일:

- [custom-template.md](../../skills/codebase-skill/writing-styles/custom-template.md)

## 커스텀 스타일 권장 흐름

1. `custom-template.md`를 복사합니다.
2. 원하는 톤, 구조, 금지 표현, 예시 문단을 작성합니다.
3. 에이전트가 그 파일을 읽습니다.
4. 에이전트는 그 내용을 `customMarkdown`으로 MCP에 전달합니다.
5. 반환된 스타일 가이드 기준으로 글을 씁니다.
6. 발행 시 `custom:<alias>` 형태로 기록할 수 있습니다.

즉, 커스텀 스타일 파일은 단순 참고 메모가 아니라 실제 자동포스팅의 기준 파일이 될 수 있습니다.
